# AI-ассистент для улучшения объявлений

Тестовое задание для стажёра Frontend.

Проект представляет собой личный кабинет продавца с доской объявлений, страницей просмотра объявления, формой редактирования и AI-ассистентом, который помогает улучшить описание, предложить ориентир по цене и ответить на вопросы по конкретному объявлению.

В качестве LLM в проекте используется **Ollama**. Для упрощения проверки предусмотрен запуск всего приложения через **Docker Compose**: клиент, сервер и Ollama поднимаются одной командой.

## Что реализовано

### Основной функционал

- страница списка объявлений `/ads`;
- поиск по названию;
- фильтрация по категориям;
- фильтр «Только требующие доработок»;
- сортировка объявлений;
- пагинация по 10 объявлений;
- переключение вида списка: `grid / list`;
- страница просмотра объявления `/ads/:id`;
- страница редактирования `/ads/:id/edit`;
- динамический набор полей формы в зависимости от категории объявления;
- сохранение изменений через `PUT /items/:id`;
- сохранение черновика формы в `localStorage`;
- отображение состояний загрузки, ошибок и пустых состояний.

### AI-функции

- генерация нового описания для объявления;
- улучшение уже существующего описания;
- определение ориентировочной рыночной цены;
- применение AI-подсказки в форму;
- чат с AI по конкретному объявлению;
- визуальное сравнение текста «Было → Стало» с подсветкой изменений.

### Дополнительные возможности

- тёмная тема с сохранением выбора в `localStorage`;
- отмена запросов при уходе со страницы;
- юнит-тесты для части бизнес-логики;
- запуск клиентской и серверной части через `docker compose`.

## Запуск через Docker Compose

Это основной сценарий запуска для проверки.

### Требования

- Docker
- Docker Compose

### Команда запуска

```bash
docker compose up --build
```

После запуска будут доступны:

- клиент: `http://localhost:5173`
- сервер: `http://localhost:8080`

### Важно про первый запуск

При первом запуске Docker Compose дополнительно скачивает модель Ollama. Поэтому первый старт может занять заметно больше времени, чем последующие.

После того как модель будет загружена, она сохранится в Docker volume и повторные запуски будут быстрее.

## Локальный запуск без Docker

### 1. Подготовить Ollama

Установить Ollama и загрузить модель:

```bash
ollama pull qwen2.5:3b
```

Если Ollama не запущена автоматически, можно запустить её отдельно:

```bash
ollama serve
```

### 2. Запустить сервер

```bash
cd server
cp .env.example .env
npm install
npm run dev
```

Сервер будет доступен по адресу:

```text
http://localhost:8080
```

### 3. Запустить клиент

```bash
cd client
cp .env.example .env
npm install
npm run dev
```

Клиент будет доступен по адресу:

```text
http://localhost:5173
```

## Реализация AI

В проекте AI-функции реализованы через отдельный серверный слой, который обращается к Ollama.

Поддерживаются три сценария:

- генерация или улучшение описания;
- получение ориентировочной цены;
- чат по объявлению с передачей контекста карточки.

По умолчанию используется модель **`qwen2.5:3b`**. Она выбрана как компромисс между качеством ответа и скоростью локального запуска. Для задач тестового задания более тяжёлая модель не требовалась.

## Принятые решения

### 1. Минимально расширен сервер

За основу взят предоставленный backend. Дополнительно были внесены минимальные изменения, необходимые для работы клиентской части и AI-функционала:

- добавлены AI-эндпоинты:
  - `POST /ai/description`
  - `POST /ai/price`
  - `POST /ai/chat`
- добавлен `GET /health` для корректного запуска через Docker Compose;
- в ответ `GET /items` включены `id` и `createdAt`, так как без них невозможно корректно построить переходы на карточку объявления и вывести дату публикации в списке.

### 2. Сортировка реализована в рамках backend-контракта

В интерфейсе поддержана сортировка по тем параметрам, которые поддерживает серверный API: по дате создания и по названию.

### 3. Пустые необязательные поля очищаются перед отправкой

Перед сохранением пустые значения (`''`) удаляются из `params`, чтобы сервер корректно валидировал запрос.

### 4. FSD использована без избыточного усложнения

Я сознательно не усложнял проект дополнительными абстракциями. Основная цель была сделать понятную структуру, которую можно быстро прочитать и поддерживать в рамках тестового задания.
